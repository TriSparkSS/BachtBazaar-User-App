## Local Cache Reset Commands

Clear Metro and file map caches from PowerShell:

```powershell
Remove-Item "$env:TEMP\metro-cache" -Recurse -Force -ErrorAction SilentlyContinue
Get-ChildItem "$env:TEMP" -Force |
  Where-Object { $_.Name -like "metro-file-map*" -or $_.Name -like "react-native-packager-cache*" } |
  Remove-Item -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "node_modules\.cache" -Recurse -Force -ErrorAction SilentlyContinue
```

Start Metro with reset cache when using a normal debug build:

```powershell
npx react-native start --reset-cache --port 8081
```

Run Android normally only after Android SDK path is fixed:

```powershell
npx react-native run-android --port 8081
```

## Portable Build Advice For Another System

On a new system, first install normal prerequisites:

- Node.js compatible with project engine: `>=22.11.0`
- Android Studio / Android SDK
- Platform tools / ADB
- Java/JDK compatible with Android Gradle Plugin

Then from the repo:

```powershell
npm install
npx react-native doctor
npx react-native start --reset-cache --port 8081
npx react-native run-android --port 8081
```

If Metro/port errors happen on a physical USB device:

```powershell
adb reverse tcp:8081 tcp:8081
```

If the project still cannot build with Gradle, check `android/local.properties` and update `sdk.dir` to the Android SDK path on that machine.
